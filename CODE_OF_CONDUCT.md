# Code of Conduct

## Our Standard

WiFiHomeView aims to be a practical, respectful open-source project. Contributors are expected to communicate clearly, assume good intent, and focus feedback on the work.

Unacceptable behavior includes harassment, personal attacks, sustained disruption, or publishing private information such as real device names, network names, hardware identifiers, credentials, or private network details.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, or block participants whose behavior makes the project unsafe or unproductive.

## Reporting

Report conduct concerns through GitHub issues only when the report can be made safely without exposing private information. If private reporting is available through GitHub, prefer that route.
